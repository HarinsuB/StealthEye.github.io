Our Research

Project Scope

Literature Survey

Fileless malware is a significant cybersecurity threat as it is entirely present in system memory and evades traditional signature-based detection methods. This paper introduces STEALTH EYE, an endpoint behavioral analysis framework for detecting fileless malware, such as ransomware, spyware, trojans, and RedLine Stealer, in real time. The framework utilizes an endpoint agent that monitors system activity in real time and captures snapshots of behavior every 60 seconds for real- time threat analysis. These captures track memory injections, DLL loading and execution, file and handle operations, service activity, process and thread behavior, registry modifications, network communications, cryptographic function usage, keystroke logging, and clipboard access. The data that is collected is analyzed through supervised machine learning mechanisms to detect patterns that indicate fileless malware activity. In contrast to traditional post-infection forensic approaches, STEALTH EYE provides real-time monitoring, notification, and active response with enhanced cybersecurity resilience against the widespread fileless attacks.

Research Gap

Limited Real-Time Detection Capabilities

Although existing studies provide valuable insights into fileless ransomware, trojans, and spyware, most focus on post-infection analysis or static detection methods. There is a critical gap in the development of proactive, real-time detection systems that can continuously monitor system behavior and detect threats as they occur.

Reactive vs. Proactive Strategies

Current detection techniques, such as memory forensics and static signature analysis, are largely reactive—responding after malware has executed. These approaches fail to provide immediate threat mitigation. There is a need for dynamic, real-time solutions that leverage machine learning to identify behavioral anomalies early and prevent attacks before damage is done.

Narrow Platform Focus

Many studies focus solely on the Windows environment or specific attack stages like persistence, overlooking malware behaviors across the full lifecycle and in cross-platform contexts. This creates a gap in developing detection methods that can generalize across diverse operating systems and network environments.

Lack of Integrated Behavioral Analysis

While research has explored rule-based systems and specific tools (e.g., YARA, MSF Venom), few integrate continuous behavioral monitoring with adaptive machine learning models. There is a pressing need for unified detection frameworks that evolve with emerging threats, especially those that use advanced evasion techniques like mimicry, process splitting, and memory-only execution.

Research Problem

Fileless malware represents a significant evolution in cyber threats, operating entirely within system memory and leveraging legitimate tools such as PowerShell, Windows Management Instrumentation (WMI), scheduled tasks, and system registries to carry out malicious activities. Unlike traditional malware, fileless variants do not leave behind identifiable files on disk, making them extremely difficult to detect using conventional signature-based or file-centric security solutions. These attacks pose a critical challenge to existing cybersecurity frameworks, which are often reactive and rely heavily on post-infection analysis or persistent file indicators. Fileless malware variants such as ransomware, information stealers, spyware, and trojans exploit the trusted nature of native system processes to execute harmful actions, including data encryption, credential harvesting, system manipulation, and unauthorized surveillance. The stealthy nature of fileless attacks allows them to bypass many traditional detection mechanisms, often blending in with normal system activity and remaining undetected until significant damage has been done. Existing security solutions frequently lack the capability to monitor real-time memory activity or identify subtle behavioral anomalies that characterize fileless threats. This research aims to address the limitations of traditional detection methods by investigating the behavioral characteristics of various types of fileless malware. The objective is to design and propose a proactive, real-time detection approach capable of identifying malicious activity based on in-memory behavior and process-level anomalies. Such an approach is essential to enhance the effectiveness of cybersecurity defenses against the growing complexity and persistence of fileless malware attacks.

Main Objective

The main objective of this research is to develop a comprehensive and proactive detection framework capable of identifying four specific types of fileless malware—ransomware, RedLine Stealer, spyware, and trojans—through real-time behavioral and memory-based analysis. The proposed system aims to detect malicious activities that occur entirely within system memory, without generating traditional file-based signatures, thereby enhancing early threat detection and improving the overall security posture against stealthy and persistent fileless attacks.

Specific Objectives

Fileless Ransomware Detection

This component focuses on identifying ransomware activity that operates without traditional file traces. The system is designed to monitor and detect behaviors indicative of in-memory ransomware execution. Detection parameters include monitoring registry changes for signs of suspicious activity, tracking abnormal thread creation and termination patterns, and detecting the creation of mutex objects commonly used by ransomware to prevent multiple instances. Additionally, the system analyzes system calls to identify patterns that deviate from normal behavior, which may indicate active ransomware processes.

Fileless RedLine Stealer Detection

This module targets the detection of RedLine Stealer, a memory-resident malware known for stealing credentials and exfiltrating sensitive data. The detection process involves monitoring memory usage to identify unauthorized code injections and tracking abnormal or unexpected DLL loading behaviors. It also includes monitoring for unusual network activity that could signal data exfiltration attempts, and observing cryptographic operations that may be indicative of efforts to bypass security protocols or encrypt stolen data.

Fileless Spyware Detection

The spyware detection component addresses threats that focus on surveillance and data harvesting through system memory manipulation. Detection strategies include monitoring keystroke logging and keyboard handle access, analyzing clipboard interactions for unauthorized usage, and detecting network-level data exfiltration patterns through injected DLLs. The system also tracks GUI interaction anomalies, memory injections, and thread handle activity. Further, it monitors callbacks to identify unauthorized code execution and identifies hidden processes that operate outside standard process listings to evade detection.

Fileless Trojan Detection

This component is designed to detect fileless trojans that use stealth techniques to execute malicious payloads via system processes. Key detection parameters involve monitoring memory usage and detecting unauthorized process injections. It also tracks abnormal DLL loading and execution attempts, observes irregularities in process and thread behavior, and monitors handle activity to identify system manipulation. Additionally, the system detects hidden processes and stealth operations used by trojans to remain undetected.

Our Solution

The STEALTH EYE system is a real-time behavioral analysis and detection framework designed to identify and mitigate four types of fileless malware: ransomware, RedLine Stealer, spyware, and trojans. The system is built around four dedicated endpoint agents, each specialized for detecting one type of fileless malware. These agents continuously monitor system activities and capture behavioral snapshots every 60 seconds. The snapshots are then analyzed by machine learning models tailored to each malware type. The system incorporates real-time data collection, feature extraction, machine learning-based classification, and alerting mechanisms to proactively detect and respond to potential threats. The system integrates four endpoint agents, each designed to detect a specific type of fileless malware: Fileless Ransomware Agent, Fileless RedLine Stealer Agent, Fileless Spyware Agent and Fileless Trojan Agent. Each endpoint agent runs with administrative privileges on a Windows environment. Its primary function is to continuously monitor system activity, capturing snapshots of key behaviors every 60 seconds. These agents focus on low-level system parameters such as memory usage, process/thread behavior, registry modifications, DLL loading, network activity, and cryptographic operations. By monitoring these behaviors, the agents can detect anomalies indicative of fileless malware, which operates entirely in memory and avoids traditional file-based signatures. Once the behavioral snapshots are collected, the data undergoes a preprocessing phase. This includes cleaning the data, normalizing formats, and handling missing values to ensure consistency across features. Feature extraction is then performed by identifying the key behavioral patterns associated with each malware type. For example, the system extracts features such as unusual DLL loading, memory manipulation, abnormal network traffic, suspicious process/thread activities, and unauthorized registry modifications. These features form the basis of the dataset used for training the machine learning models. The extracted features are used to train four separate machine learning models, one for each malware type: ransomware, RedLine Stealer, spyware, and trojans. The models are trained using previously collected labeled datasets of system behavior, where each sample is categorized as either benign or malicious. Various machine learning algorithms, such as Logistic Regression, Decision Trees, Support Vector Machines (SVM), and ensemble methods, are evaluated during the training process. Based on performance metrics such as accuracy, precision, recall, and F1-score, the best-performing model for each malware type is selected. For example, the Random Forest classifier is commonly used for fileless ransomware, Gradient Boosting is effective for RedLine Stealer, and Random Forest is also used for detecting spyware. To address data imbalance issues, techniques like SMOTE (Synthetic Minority Over-sampling Technique) and under-sampling are applied during training to ensure that both benign and malicious samples are fairly represented. Once the models are trained, they are serialized and embedded into the corresponding endpoint agents. During real-time operation, each agent captures and processes new behavioral snapshots locally and evaluates them using the trained models. The models compute a probability score indicating the likelihood of malware activity. A custom threshold (e.g., 85%) is applied to minimize false positives and ensure that only high-confidence detections trigger alerts. When the threat probability exceeds the defined threshold, the system generates an alert, which is logged in a dedicated "alerts.json" file. The alert, along with the associated behavioral snapshot and metadata, is securely transmitted to the security team via Secure File Transfer Protocol (SFTP), ensuring the confidentiality and integrity of the data. The system includes a centralized administrative dashboard, which visualizes the incoming threat data from all endpoint agents. The dashboard displays key information such as detection timestamps, threat levels, malware categories, and contributing behavioral features. This interface enables security teams to monitor threats in real time, trace malicious behavior patterns, and take swift action when necessary. The dashboard aggregates data from all the endpoint agents, providing a comprehensive view of the status of the entire network. Security teams can analyze the threat data, investigate the behavior patterns, and take appropriate countermeasures, all from one centralized interface.
